\noindent \textbf{Question:} Write \verb|Makefile| for the complex number program containing three files: \verb|complex.hpp|, \verb|complex.cpp| and \verb|main.cpp|. \verb|Makefile| should first create object files and then link them.\\
\begin{lstlisting}[language=bash,caption={Makefile}]
# Makefile
binfile: complex.o main.o
	g++ complex.o main.o -o binfile

complex.o: complex.cpp complex.hpp
	g++ -c complex.cpp -o complex.o

main.o: main.cpp complex.hpp
	g++ -c main.cpp -o main.o
\end{lstlisting}
Note that \verb|complex.cpp| includes \verb|complex.hpp| file. If \verb|complex.hpp| file does not exist then \verb|complex.cpp| cannot be compiled. That's why \verb|complex.o| depends on both \verb|complex.cpp| and \verb|complex.hpp| files. Similarly \verb|main.cpp| also includes and is dependent on \verb|complex.hpp| file.

Although including header file as dependency is not a strict requirement but it is a good practice to check for the existence of header files.
